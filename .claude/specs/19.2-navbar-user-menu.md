# Spec: Navbar User Menu

## Roadmap description
Replaces the username text and standalone Logout link in the navbar with a user icon that opens a dropdown on desktop and swaps the hamburger icon on mobile.

## Overview
The current navbar displays the logged-in user's name as a text link alongside a separate "Logout" link, which clutters the navigation bar and does not follow modern UX patterns. This release replaces the username text with a Lucide `user` icon in the desktop nav. Clicking the icon opens a dropdown menu containing "My Profile" and "Log Out". On mobile (≤600px), the hamburger icon is replaced by the same user icon when logged in — tapping it opens the existing mobile slide-down menu with relabelled items ("My Profile" and "Log Out"). When logged out, the hamburger icon remains unchanged. No new routes or database changes are required — all changes are in `base.html`, `style.css`, and `main.js`.

## Release context
Release 2 of 2 — Navbar User Menu. This release modernises the navbar with a user icon and dropdown on desktop, and a user icon replacing the hamburger on mobile when logged in. Release 1 (Responsive Profile Layout) is already shipped and is independent of this work. No open questions remain — the processed thought and release plan fully specify the behaviour.

Deferred/out of scope:

- `<details>/<summary>` no-JS fallback for the desktop dropdown — the app already depends on JS for modals and analytics

## Error handling
No new error cases — this release only changes navbar UI. All existing auth guards, flash messages, and redirect behaviours remain unchanged. The dropdown must gracefully handle:
- Clicking outside the dropdown closes it without navigating
- Pressing Escape closes the dropdown
- Rapid clicking does not cause visual glitches

## UI/UX notes
- The user icon must be the Lucide `user` icon (consistent with the existing icon set loaded via CDN)
- Desktop dropdown appears below the user icon on click; clicking again or clicking outside closes it
- The dropdown contains two items: "My Profile" (links to `/profile`) and "Log Out" (links to `/logout`)
- On mobile (≤600px) when logged in: the hamburger button is replaced by a user icon button that toggles the existing mobile menu
- On mobile (≤600px) when logged out: the hamburger icon remains unchanged
- Mobile menu items are relabelled: username text → "My Profile", "Logout" → "Log Out"
- The dropdown must have sufficient contrast (ink text on paper-card background) and a subtle border/shadow to distinguish it from the nav bar
- All interactive elements must have hover and focus-visible states using existing CSS variables
- Accessibility: the user icon button must have `aria-expanded` and `aria-controls` attributes; the dropdown must have `role="menu"`; menu items must be keyboard-navigable

## Rules for implementation
- No SQLAlchemy or ORMs — not applicable (no DB changes)
- Parameterised queries only — not applicable (no DB changes)
- Use CSS variables — never hardcode hex values
- All templates extend `base.html`
- Preserve all existing routes, permissions, and behaviours
- The Lucide icon library is already loaded via `<script src="https://unpkg.com/lucide@latest"></script>` in `base.html` — use `data-lucide="user"` attributes and call `lucide.createIcons()` after DOM changes
