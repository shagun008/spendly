# Spec: Oxos Profile Page MVP

## Roadmap description
A central dashboard page for the Oxos platform showing business capabilities and best practices for authenticated users.

## Overview
This release creates the Oxos Platform page at `/platform`, which serves as the default landing page for the application. The page displays two sections: Business Outcomes (Reports and Business Applications cards) and Learnings (top 5 best practices from CLAUDE.md). Users can navigate to the Expense business application which opens the existing profile page in a new tab. The design follows existing card-based patterns with responsive layout. The old Spendly landing page is moved to `/expense-home`.

## Release context
Release 1 of 2 for feature 21. This release provides the core Oxos Platform page as the default landing page with Business Outcomes and Learnings sections. The Context section with card flip animation is deferred to Release 2.

## UI/UX notes
- Full navbar reused from existing application
- Brand name updated to "Oxos" in navbar
- Footer name updated to "Oxos"
- Cards follow existing `.feature-card` pattern from style.css
- Hover effects using existing design system (color transitions, elevation)
- Responsive grid layout that collapses to single column on mobile
- Expense card: opens in new tab via `target="_blank" rel="noopener"`
- Learnings display as a clean list with icons, titles, and descriptions
- All cards use `--paper-card`, `--border`, `--radius-md`, and existing color variables

## Rules for implementation
- No SQLAlchemy or ORMs — use raw sqlite3/psycopg2 queries only
- Parameterised queries only
- Passwords hashed with werkzeug — `generate_password_hash` / `check_password_hash`
- Use CSS variables — never hardcode hex values
- All templates extend `base.html`
- Currency displayed in ₹ — format amounts as `₹{amount:,.2f}`
- Dates stored as `YYYY-MM-DD` strings — validate with `datetime.strptime(value, "%Y-%m-%d")`
