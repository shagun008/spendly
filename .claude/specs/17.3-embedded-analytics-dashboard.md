# Spec: Embedded Analytics Dashboard

## Roadmap description
Adds spending charts to your Profile page so you can see trends, category breakdowns, and monthly comparisons without navigating away.

## Overview
This release embeds an interactive Analytics dashboard between the profile card and the filter bar on the Profile page. The current `/analytics` route is a "Coming Soon" placeholder; this release replaces it with a real, data-driven dashboard. Users get three switchable chart views — spending trends (line chart), category breakdown (pie/bar chart), and monthly comparisons — powered by Chart.js loaded via CDN. The standalone Analytics nav item, route, and template are removed as part of consolidating Profile as the central hub.

## Release context
Release 3 of 3 — the final release of the User Profile Page Updates feature. This release depends on Release 2 (Quick Add Expense Modal) which already removed the standalone Add Expense nav item and route. This release performs the analogous cleanup for the Analytics placeholder.

**In scope:**
- Embedded Analytics section on the Profile page with three chart views (spending trends, category breakdown, monthly comparisons)
- Chart.js loaded via CDN (pinned stable version)
- Remove standalone `/analytics` route from `app.py`
- Delete `templates/analytics.html`
- Remove Analytics nav item from `base.html` (both desktop and mobile)

**Deferred (out of scope for this feature):**

**Open questions from discovery:**

## Routes
- `GET /profile` — modified to include analytics data (trends, monthly comparison) alongside existing stats/transactions/categories. No new route needed.
- `GET /analytics` — **deleted** (placeholder route removed)

## Database changes
No new tables or columns. New query functions will be added to `database/queries.py`:
- `get_spending_trends(user_id, date_from, date_to)` — daily spending totals for line chart
- `get_monthly_comparison(user_id)` — current month vs previous month totals and category breakdown

These read from the existing `expenses` table using parameterised queries.

## Error handling
- User not logged in: auth guard on `/profile` redirects to `/login` (existing behaviour)
- No expenses yet: charts show empty state with a message like "Add expenses to see your analytics"
- Chart.js fails to load: dashboard section shows a fallback message; does not break the rest of the profile page
- Date filter applied: analytics data respects the current date range (if feasible within scope; otherwise analytics shows all-time data and a note is added to the spec)

## UI/UX notes
- Dashboard appears between the profile card and the filter bar on the Profile page
- Three clickable tabs: "Trends", "Categories", "Monthly Comparison" — active tab highlighted with `--accent` colour
- Charts use the existing colour palette: `--accent` for primary, `--accent-2` for secondary, `--ink-muted` for axes/grid
- Chart containers must be responsive: full width on mobile, max 2/3 width on desktop within the profile grid
- Empty state: when user has no expenses, show a centered message with an icon (Lucide `bar-chart-2`) and text "Add expenses to see your analytics"
- All form elements have labels; tabs are keyboard-navigable (`role="tab"` + `aria-selected`)
- Flash messages not required for this release (no user-triggered actions beyond tab switching)

## Rules for implementation
- No SQLAlchemy or ORMs — raw `sqlite3`/`psycopg2` queries only
- Parameterised queries only — never interpolate user data into SQL
- Use CSS variables — never hardcode hex values
- All templates extend `base.html`
- Auth guard on every protected route
- Ownership check on all expense queries — must include `AND user_id = %s`
- Currency displayed in ₹ — format as `₹{amount:,.2f}`
- Dates stored as `YYYY-MM-DD` strings
- Chart.js loaded via CDN, not bundled
