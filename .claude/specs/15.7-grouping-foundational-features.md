# Spec: Grouping Foundational Features

## Roadmap description
Groups the first 10 features under a collapsible "Foundational Features" header and makes parent features expand to show their releases.

## Overview

The roadmap page currently renders all features and releases in a flat list. Clicking a feature row opens a description detail card below it. This enhancement introduces two levels of hierarchy:

1. **Group row:** Features 01–10 are wrapped under a single "Foundational Features" group row with a chevron. The group is collapsed by default. Clicking it toggles visibility of all child rows.
2. **Parent feature accordion:** Parent feature chevrons (e.g. feature 11, 12, 15) no longer open a description card — instead they toggle their child release rows in an accordion pattern (only one parent expanded at a time). The description detail row is removed entirely.

No backend or database changes are needed — `get_all_features()` already returns `parent_number` on every row, and the features table already has the correct hierarchy. Changes are limited to `templates/roadmap.html`, `static/css/roadmap.css`, and inline JS.

## Error handling

- If no features are returned from the DB, the existing `{% else %}` block already shows "No features yet." — no change needed.
- If a feature has no `parent_number` (top-level), it renders as a normal row — no group or accordion behavior applied.

## UI/UX notes

- **Group row:** Renders as a full-width row with a chevron icon on the left, "Foundational Features" label, and a count badge showing "10 features". Styled distinctly from feature rows (slightly bolder, different background).
- **Collapsed state:** Child rows under the group are hidden (`display: none`). Chevron points right.
- **Expanded state:** Child rows are visible. Chevron rotates 90° (points down).
- **Parent feature accordion:** Clicking a parent feature chevron expands its child release rows. If another parent is already expanded, it collapses first (accordion behavior). Chevron rotates 180° on expanded parent features (reusing existing `.roadmap-row--expanded .roadmap-chevron` CSS).
- **Release rows under a parent:** When parent is collapsed, release rows are hidden (`display: none`). When expanded, release rows are visible.
- **Description detail row removed:** The `<tr class="roadmap-detail-row">` and its card are completely removed from the template. Feature descriptions are no longer shown on the roadmap page.
- **Flash messages:** Not applicable — this is a read-only page with no form submissions.
- **Accessibility:** Group row and parent feature row both use `aria-expanded` and `aria-controls` attributes. Chevron icons are decorative (`aria-hidden`). Focus management is not required (click-only interaction).
- **Responsive:** Group row follows the same mobile breakpoints as existing rows. On mobile (≤600px), the same columns are hidden as the existing responsive CSS specifies.
