# Spec: Roadmap Detail View

## Roadmap description
Makes the roadmap interactive — clicking any feature or release row expands a detail card inline showing its description. No page reload needed.

## Overview
The `/roadmap` pipeline table (shipped in 15.1) shows each feature as a row of stage dots, but gives no way to learn more about any individual feature. This release adds an expand-in-place detail card: clicking a feature row toggles a detail panel immediately below it — showing the feature number, title, status badge, description, and release breakdown (child releases with their own stage dots). No page reload or separate route is needed. It is a pure front-end addition: vanilla JS toggle, new CSS rules in `roadmap.css`, and a small template change to `roadmap.html` to embed the detail row data.

## Database changes
The `features` table already has a `description` column (TEXT, nullable). No schema changes are needed.

One query change: `get_all_features()` currently omits `description` from the SELECT. It must be added so the template can render it in the detail card.

`_feature_row()` must also be updated to pass `description` through in its return dict.

## Templates
- **Modify:** `templates/roadmap.html`
  - Add a `data-feature-number` attribute to each `<tr class="roadmap-row">` for top-level feature rows
  - Add a hidden detail `<tr class="roadmap-detail-row">` immediately after each top-level feature row, containing the detail card HTML
  - Detail card structure: feature number + title heading, status badge, description paragraph (or "No description." if null), and a sub-table of child release rows (filtered by `parent_number`)
  - Add `aria-expanded` on the clickable row; toggle it with JS
  - Release sub-rows inside the detail card show their own stage dots (same dot markup as the main table)

## Error handling
- If `description` is NULL in the DB, the detail card renders "No description." — never render `None` or an empty string
- If a feature has no child release rows, the detail card omits the releases sub-table and shows nothing in its place (no "No releases" message needed)
- JS must be defensive: if a clicked row has no matching detail row in the DOM, do nothing (no JS errors)

## UI/UX notes
- **Interaction:** Click anywhere on a top-level feature row to expand; click again to collapse. Only one row expanded at a time — opening a new row collapses any previously open one
- **Toggle indicator:** Add a small chevron icon (Lucide `chevron-down`, loaded via CDN) to the feature title cell. Rotate it 180° when the row is expanded
- **Expanded row:** `.roadmap-detail-row` spans all columns (`colspan` = number of `<th>` elements = 9). The card sits flush under the parent row with a subtle left border in `--accent` colour
- **Detail card layout:** Two-column layout on desktop (left: metadata, right: description); stacks to single column on mobile (`@media (max-width: 600px)`)
- **Release sub-table:** Rendered as a compact table inside the card — same column headers as the main table but smaller font. Only shown if the feature has child rows
- **Cursor:** `cursor: pointer` on clickable rows; `cursor: default` on release sub-rows (they are not clickable)
- **Animation:** `max-height` CSS transition (0 → auto via JS class swap) for a smooth slide-down. Keep it subtle (~200ms ease)
- **Accessibility:** `aria-expanded="false"/"true"` on the clickable `<tr>`; `aria-controls` pointing to the detail row's `id`. Detail row has `role="region"` and `aria-label` = feature title
- **No flash messages** — this is a read-only public page
- **Responsive:** Detail card must not overflow on mobile; use `overflow-x: auto` on the inner release sub-table if needed
- **Colour:** Use CSS variables only — `--accent` for the left border, `--paper-warm` for the card background, `--border-soft` for inner dividers

## Rules for implementation
Follow all rules in `CLAUDE.md`:
- No SQLAlchemy or ORMs — raw `psycopg2` queries only
- Parameterised queries only
- Use CSS variables — never hardcode hex values
- All templates extend `base.html`
- No auth guard on `/roadmap` — it is intentionally public
- Vanilla JS only — no jQuery or other libraries
