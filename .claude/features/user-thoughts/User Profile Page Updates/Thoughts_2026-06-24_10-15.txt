User Profile Page Updates

This enhancement should focus on making expense management faster and more intuitive directly from the Profile page.

1. Quick Edit Expense from Profile
- Replace the existing Edit Expense functionality – instead of opening a new page on clicking 'Edit Expense', Edit Expense should open a modal containing the same fields and functionality currently available on the Edit Expense page
- Upon submission, the expense should be saved directly to the user's expenses list
- It would also be valuable to explore additional ideas for making editing expense faster and more intuitive, such as inline editing, keyboard shortcuts, smart defaults, or recently-used category suggestions
- Once this functionality has been successfully updated, delete any associated pages, routes, components, and unused code

The end goal is to reduce friction when users need to correct or update an expense, keeping them in the flow of reviewing their expenses without navigating away from the Profile page.
