# Processed Thought: Release Notes Modal

## Problem / Goal
The roadmap shows WHEN each pipeline stage happened but not WHAT was found or done. Clicking a completed (green) stage dot should open a modal showing the full test report and code review report for that release — turning the dots into a full audit trail.

## Who benefits
Visitors to the public roadmap (developers, potential users) who want to see the quality bar applied to each release.

## Success looks like
Clicking a green dot on a release sub-row opens a modal with the test and review reports. Modal is dismissible via click-outside or Escape. Non-clickable when no reports exist. Parent feature row dots don't trigger it.

## Constraints, risks, dependencies
Requires two new TEXT columns on the features table (test_report, review_report). Harness commands (/test-feature, /code-review-feature) must be updated to write to these columns as part of their stamp step. Data must survive reseeds.

## Implementation ideas / open questions
Store test_report and review_report in the features DB table. Harness stamp steps write reports there. UI: clicking green dot fetches the row data and renders a modal. Modal title: "<number> — <title>". Show whatever reports are available; dots non-clickable if neither exists.

Not specified.
