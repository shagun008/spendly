# Processed Thought: Roadmap Stage Metrics

## Problem / Goal
Track how long each pipeline stage takes and how many tokens it consumes, and surface that data on the roadmap page dot tooltips.

## Who benefits
Developer / project owner — visibility into how much effort each stage costs in time and tokens.

## Success looks like
Each filled dot on the roadmap shows time taken (in hours) and tokens used on hover. Data is stored as start/end time + start/end token count columns on the features table, one pair per stage (7 stages × 4 columns = 28 new columns).

## Constraints, risks, dependencies
Large schema addition — 28 new columns on the features table. Harness wiring (writing the values at runtime) is a separate concern, likely deferred to 15.3 or a dedicated sub-release. Depends on 15.1 (features table exists) and 15.2 (dot tooltip infrastructure exists).

## Implementation ideas / open questions
New columns on the features table per stage:
- `captured_start_at`, `captured_end_at`, `captured_start_tokens`, `captured_end_tokens`
- `planned_start_at`, `planned_end_at`, `planned_start_tokens`, `planned_end_tokens`
- `spec_start_at`, `spec_end_at`, `spec_start_tokens`, `spec_end_tokens`
- `implemented_start_at`, `implemented_end_at`, `implemented_start_tokens`, `implemented_end_tokens`
- `tested_start_at`, `tested_end_at`, `tested_start_tokens`, `tested_end_tokens`
- `reviewed_start_at`, `reviewed_end_at`, `reviewed_start_tokens`, `reviewed_end_tokens`
- `shipped_start_at`, `shipped_end_at`, `shipped_start_tokens`, `shipped_end_tokens`

The tooltip in roadmap.html computes duration (end − start in hours) and token delta (end_tokens − start_tokens) and appends them to the existing hover text.

Open question: should the tooltip show both metrics only when both are available, or show whichever is present?
