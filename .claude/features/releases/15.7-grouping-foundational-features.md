# Release Plan: Grouping Foundational Features

## Roadmap description
Groups the first 10 features under a collapsible "Foundational Features" header and makes parent features expand to show their releases.

## Summary
The roadmap page currently renders all features and releases in a flat list with a description card that opens on click. This enhancement adds a "Foundational Features" group row that collapses features 1–10 under a single header, and repurposes the parent feature expand/collapse behavior to show child release rows instead of a description card. The description detail row is removed entirely. Changes are limited to `templates/roadmap.html`, `static/css/roadmap.css`, and inline JS — no backend or database changes are needed.

## Releases

### Release 1 — Grouping Foundational Features
- **Scope:** Add "Foundational Features" group row (chevron, collapsed by default) wrapping features 01–10; repurpose parent feature chevron to toggle child release rows (accordion — one parent open at a time); remove description detail card; add `data-parent-group` and `data-parent` attributes to template rows; update JS toggle logic for group + parent expand/collapse; add `.roadmap-group-row` CSS; remove deprecated `.roadmap-detail-row` / `.roadmap-detail-card` CSS
- **Spec slug:** grouping-foundational-features
- **Spec arg:** `15.7 grouping-foundational-features`
- **Depends on:** 15.2 — Expand-in-Place Detail View (current expand/collapse JS and detail row structure)
- **Risk:** low

## Deferred / Out of scope
- Expanding the grouping pattern to other feature ranges beyond 1–10 (e.g. grouping features 11+)
- Persisting collapse/expand state across page reloads

## Open questions
Resolved:
1. **Accordion for parent features:** Only one parent feature expanded at a time
