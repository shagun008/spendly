❯ How to setup SSH keys in Github account?                                      

⏺ Here's how to set up SSH keys for GitHub:

  Step 1: Generate an SSH key

  Run this in your terminal:
  ssh-keygen -t ed25519 -C "your_email@example.com"
  Press Enter to accept the default file location, then set a passphrase (or
  leave blank).

  Step 2: Copy the public key
  cat ~/.ssh/id_ed25519.pub
  Copy the entire output.

  Step 3: Add to GitHub
  1. Go to GitHub → Settings → SSH and GPG keys
  2. Click New SSH key
  3. Give it a title (e.g. "My Laptop")
  4. Paste the key into the Key field
  5. Click Add SSH key

  Step 4: Test the connection
  ssh -T git@github.com
  You should see: Hi shagun008! You've successfully authenticated...

  Step 5: Push your code
  git push -u origin main

  ---
  You can run the commands directly in your terminal by typing ! <command> in
  this prompt, e.g.:
  ! ssh-keygen -t ed25519 -C "your_email@example.com"

❯ How to resolve this error                                                     

⏺ "Host key verification failed" means GitHub's SSH host key isn't in your
  ~/.ssh/known_hosts file. Fix it by running:

  ssh-keyscan github.com >> ~/.ssh/known_hosts

  Then try pushing again:
  git push -u origin main

  ---
  If that still fails, verify your SSH key is loaded and GitHub can authenticate
   you:
  ssh -T git@github.com

  Expected output: Hi shagun008! You've successfully authenticated...

  If you get "Permission denied", your SSH key hasn't been added to GitHub yet —
   follow the setup steps from my previous message.
